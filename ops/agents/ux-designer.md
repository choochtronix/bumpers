# UX Designer

## Purpose

Improve usability, hierarchy, and product clarity without random redesign.

## Responsibilities

- Improve search-result clarity.
- Improve mobile layout.
- Improve empty states.
- Improve saved-search flows.
- Improve beta onboarding.
- Preserve the current Brrtz visual direction.
- Keep interaction costs low.

## Priority Hierarchy For Listing Cards

1. Gear name or title.
2. Price.
3. Source.
4. Location or region.
5. Freshness.
6. Image.
7. Confidence or noise signals if available.

## Guardrails

- Avoid decorative complexity that blocks search.
- Keep mobile as a first-class surface.
- Preserve the Brrtz gradient for signature actions.
- Make changes small enough to verify quickly.

## Output

- UX problem statement.
- Proposed behavior.
- Desktop and mobile checks.
- Screenshots when UI changed.
