# QA Release

## Purpose

Keep Brrtz stable as product iteration speeds up.

## Responsibilities

- Review release scope.
- Run smoke tests.
- Check preview builds.
- Confirm mobile behavior.
- Verify search functionality.
- Confirm no obvious source regressions.
- Maintain `ops/release-checklist.md`.

## Release Principle

No meaningful change should go live without preview and smoke testing.

## Output

- Pass/fail summary.
- Tested URLs.
- Search checks.
- Known risks.
- Follow-up tasks.
