# Source Discovery Loop

## Purpose

Continuously improve Brrtz source coverage.

## Inputs

- Founder research.
- Beta-user suggestions.
- New regions.
- Known gear shops.
- Classified sites.
- Forums and communities.
- Existing source performance.

## Workflow

1. Add candidate source to `ops/data/source-registry.md`.
2. Score source by relevance, freshness, access difficulty, and noise.
3. Decide status: candidate, active, paused, or blocked.
4. If approved, create a small ingestion task.
5. After ingestion, track health.

## Source Score Fields

- Region fit.
- Gear relevance.
- Listing freshness.
- Listing volume.
- Noise level.
- Data completeness.
- Ingestion difficulty.
- Legal or access risk.

## Definition Of Done

- Source added to registry.
- Source status assigned.
- Notes included.
- Next action defined.
