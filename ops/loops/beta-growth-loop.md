# Beta Growth Loop

## Purpose

Convert beta use into product learning.

## Inputs

- Beta-user feedback.
- Invite requests.
- Saved searches.
- Repeated failed searches.
- Missing gear categories.
- Missing sources.
- Region requests.

## Workflow

1. Collect feedback.
2. Sort into categories:
   - bug
   - source gap
   - relevance issue
   - UX issue
   - feature request
   - region request
3. Rank by frequency and product importance.
4. Convert top issues into Codex tasks.
5. Add summary to beta-feedback digest.

## Definition Of Done

- Feedback summarized.
- Tasks created.
- Non-priority requests parked.
- Next beta improvement selected.
