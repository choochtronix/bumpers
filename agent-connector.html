Draft connector spec

Brrtz can become an agent-callable gear layer.

This page defines the intended connector shape before implementation: tool names, user permissions, boundaries, and the minimum useful workflows for ChatGPT, Claude, and other MCP-capable agents.

Machine-readable draft Agent brief llms.txt

Current status

This is a draft planning spec. The public beta supports shareable Brrtz search URLs today. The connector tools below are not live yet and should be treated as the intended contract for a post-beta MCP server.

Candidate tools

Tool Purpose Permission Level
search_gear Search Brrtz by terms, region, category, source selection, max price, and exclude terms. Read-only
get_regions Return supported region IDs, labels, currencies, status, and source coverage. Read-only
get_sources Return source IDs, labels, regions, access posture, and whether a source is live or assist-style. Read-only
save_search Create or update a saved search intent for a signed-in user. User-authorized write
watch_listing Add a specific listing URL or normalized listing ID to the user's watchlist. User-authorized write
get_new_matches Return new or fresh matches for a saved search since the last user-visible scan. User-authorized read

Permission posture

Read-only discovery should be available without modifying user data. Saved searches, watched listings, alerts, profile preferences, and notification workflows should require explicit user authorization and should never expose private saved-search data to other users.

Listing boundaries

Brrtz should return normalized listing summaries and source links, not republish full marketplace databases. The connector should preserve source attribution, region, currency, original URL, and freshness context for every listing.
