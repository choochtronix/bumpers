# Brrtz Design System — Spec & Workflow

**Version 1.0.0 · 2026-07-05**
Audience: humans and agents (Codex, Claude, future collaborators).
Visual authority: the Claude Design project "Design system integration for brrtz". Engineering authority: Codex, in the `bumpers` repo.

---

## 1. The contract

There is exactly ONE source of visual truth: **`tokens/brrtz-tokens.css`** (mirrored as `tokens/brrtz-tokens.json` for programmatic reads).

- **Claude (design)** owns token values, component specs, and this document. Brand pivots happen here first.
- **Codex (engineering)** consumes tokens. It never invents or hardcodes visual values.

### Rules for Codex (add to AGENTS.md)

```markdown
## Design System Contract
- `design-system.css` is generated by the Brrtz design system. NEVER edit it by hand.
- NEVER hardcode colors, font families, font weights, radii, shadows, or
  transition durations in styles.css, inline styles, or JS. Reference the
  CSS variables from design-system.css. If no variable fits, use the nearest
  one and flag it — do not invent a value.
- Signature brand values that MUST come from variables:
  #ff00ff → var(--color-brand-hot), gradients → var(--brand-gradient),
  22px card radius → var(--radius-card-large), 160ms → var(--motion-standard).
- Dark theme is `body[data-theme="dark"]`. Never write per-theme colors in
  component CSS; theme switching must work purely through the variables.
- When adding a new marketplace source, request a `--source-<id>` token
  instead of hardcoding its color.
```

## 2. The sync loop (brand pivot → live site)

1. Brand change is made in the Claude design project (e.g. magenta → orange).
2. Claude regenerates `brrtz-tokens.css` + `brrtz-tokens.json` and updates the spec doc.
3. You copy both files into the repo: `brrtz-tokens.css` **replaces `design-system.css`** (keep the filename `design-system.css` so no HTML changes are needed); `brrtz-tokens.json` lives at repo root or `/tokens`.
4. Tell Codex: *"Design tokens updated — verify no regressions and hunt any hardcoded values that should now use tokens."*
5. Site repaints. No component CSS changes required — that's the point.

The file is a strict superset of the current `design-system.css` (same variable names + compat aliases), so it is drop-in safe today.

### Local token checks

After updating the token JSON or generated CSS, run:

```sh
npm run design:sync
npm run design:check
```

`design:sync` keeps the marketplace source-token namespace in `design-system.css`
aligned with `tokens/brrtz-tokens.json`. It does not replace the external design
system generator. `design:check` catches unresolved custom properties, unresolved
source-token references, duplicated AEO brand colors, and AEO pages that fail to
load the shared token layer before `aeo.css`.

## 3. Architecture

Three layers inside the token file:

1. **Brand layer** (`--brrtz-*`) — magenta, blue, ink, paper, carbon, font. ~6 decisions. A rebrand or sub-brand touches only these + theme overrides.
2. **Semantic layer** (`--color-*`, `--radius-*`, `--shadow-*`, `--motion-*`, `--text-*`, `--space-*`, `--weight-*`, `--source-*`) — what components actually reference.
3. **Compat aliases** (`--bg`, `--ink`, `--panel`…) — what today's `styles.css` uses. Frozen; new code should prefer the semantic layer.

**Sub-brands later:** add a scope like `body[data-brand="x"]` that re-binds brand-layer variables, same pattern as the dark theme.

## 4. Brand summary

- **Wordmark:** heavy blackletter-inflected "brrtz" (`assets/ICONS-site/brrtz_logo.svg`), always rendered via CSS mask + `currentColor` so it inherits theme ink.
- **Signature:** magenta→blue gradient `linear-gradient(110deg, #ff00ff, #0072ff)`; hot magenta `#ff00ff` as the single loud accent (watch-active, hover borders, loading text).
- **Voice of the UI:** retro-futurist catalog utility — near-mono chromatic surfaces, very heavy type (weights 650–950), uppercase micro-labels, big soft card shadows, perspective line-graph motifs in loading states.
- **Type:** Geist only. Weights: 650 medium / 800 bold / 850 heavy / 900 black / 950 display.
- **Themes:** dark is the app default; light fully supported. Marketplace source colors are theme-invariant.

## 5. Known debt (for Codex cleanup passes)

`styles.css` currently bypasses the token layer in many places. Highest-value fixes:

- `#ff00ff` hardcoded (watch button active, aeo hover borders, loading copy) → `var(--color-brand-hot)`
- `#111416`/`#111418`/`#686868` and other one-off inks/greys → nearest semantic token
- `border-radius: 22px` on `.listing-card` → `var(--radius-card-large)`; `16px` list rows → `var(--radius-card-list)`
- literal `160ms ease` transitions → `var(--motion-standard)`
- `.open-link` and `.watch-button` hardcode white backgrounds even in dark theme → tokens
- `aeo.css` duplicates the gradient and greys → tokens
- Naming: repo mixes `bumpers` and `brrtz` (`bumpers_logo.svg` = `brrtz_logo.svg`). Pick `brrtz` for all new work.
